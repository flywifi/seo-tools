# 29. P44 Background Updating

- Date: 2026-07-12
- Status: Accepted

## Context

The user asked to keep Claude and ChatGPT users updated in the background to the fullest extent each modality allows, for non-technical users, while keeping private data local and never touching GitHub on their behalf. The research showed the honest ceilings differ sharply by surface (true-background only via plugin auto-update or a hosted connector; pasted knowledge is always a frozen re-upload; consumer GPT/Project knowledge has no write API), so the plan builds a read-only consent-gated check, a reversible consent-first data-migration path, and per-surface distribution guidance, tagging every unverifiable vendor claim NEEDS VERIFICATION rather than asserting it.

## Decision

Built the update story for an established user across Claude and ChatGPT, keyed to what each surface can actually do, verified against current (2026-07-12) vendor docs by an adversarial research pass. Self-update CHECK: tools/update_check.py polls the repo's own GitHub releases (reusing dependency_currency._http_get_json/fetch_latest/parse_version and version.read_versions) and reports current/behind/ahead against VERSION; it never pulls or writes code. Seeds a creator-os-release currency entry plus new Anthropic/OpenAI/MCP surface sources via seed-sources. NOTIFY: tools/update_notify.py gates the check behind an opt-in background_update_check capability (off by default; a headless/cron run with the flag off makes no network call) and emits ONE passive notice only when behind, always pointing at the explicit tools/update.py; FRESHNESS.md reconciles this with the never-nag rules (a user-enabled, non-repeating notice is a chosen setting, not a nag). Surfaces: wizard /updates screen (version, one-click enable, per-surface guidance) and MCP get_server_info + update_check tools plus a quiet startup line. DATA SAFETY (consent-first): tools/local_audit.py is the read-only schema audit (one quiet line, details on demand, writes nothing); CHANGELOG.migrations.json carries a human-authored why_it_matters + concrete_impact per data-shape schema_version (no-fabrication) and drift invariant 33 requires an entry for every pipeline template's current schema_version; tools/migrate_local.py provides the safe read-time compat_view default (missing fields read as null, existing values untouched) and an opt-in, per-file/per-field consented, reversible write migration (backup + atomic write + byte-for-byte rollback to gitignored .local logs). DISTRIBUTION: the git-backed .claude-plugin marketplace gives Claude Code/Cowork background updates (documented: third-party marketplaces default auto-update off; Cowork org Sync-automatically fires only on a version-bump PR merge; the Cowork sandbox is ephemeral so .local data must be externalized). Hosted remote-MCP is the only true-background browser path: the mcp-connector runbook and CROSS-MODALITY state the rule that the tool set must stay small and stable (content flows through responses) because neither claude.ai nor ChatGPT reliably picks up a changed tool contract mid-session, with serverInfo.version as the poll signal and OAuth 2.1 (RFC 9728/8707/PKCE) deployer requirements. ChatGPT: TRANSITIONS documents frozen paste (re-upload; no management API) vs a stable Action/dev-mode connector (server-side live) vs agent mode (prep-only, human pastes). docs/UPDATING.md is the consolidated per-surface runbook.

# seasonal-trends artifact types

## Seasonal content plan
A pillar-mapped idea cluster with trend verification and a publish schedule for a seasonal window. Required elements: window name and peak dates, idea cluster (each idea: working_title, pillar, persona_served, seasonal_urgency, trend_status from trend-check or "not_checked"), publish schedule from calendar-slot (one date per idea), retrieval_gaps list, and a govern-artifact gate result.

## Seasonal map result
A window mapping for one topic. Required elements: seasonal_type (evergreen/seasonal/recurring_annual), peak_window (e.g., "September to October"), publish_by (e.g., "August 20 to 31"), urgency (immediate/upcoming/plan_ahead/off_season), and rationale.

## Trend-checked idea
One idea with live trend momentum status. Required elements: working_title, pillar, trend_status (rising/flat/declining/unknown), trend_source (retrieval or not_checked), stale flag if data is older than 14 days, and a note if trend-check was skipped.
